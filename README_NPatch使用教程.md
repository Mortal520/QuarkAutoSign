# 夸克扫描王自动签到 - Xposed模块使用教程

## 当前功能

- 自动签到：每天最多触发 1 次
- 自动抽奖：每天自动尝试 3 次（可开关）
- Hook 成功可视化：夸克启动后弹 Toast 提示
- 运行日志：持久化保存，模块内可直接查看
- 注入确认入口：夸克设置/个人相关页面注入“已注入”入口

## 前置条件

1. 已安装 `NPatch` 或 `FPA`
2. 已安装夸克扫描王并登录账号
3. 已安装本模块 APK

## 安装与启用

1. 在 NPatch/FPA 中启用模块
2. 对夸克扫描王执行注入/修补
3. 安装修补后的夸克扫描王
4. 打开夸克扫描王，观察是否出现 Toast：`[夸克签到] Hook成功，模块已激活`

## 日志与状态查看

### 方式1：模块 App 内查看（推荐）

打开桌面图标 `夸克自动签到`：

- 最近签到历史（含来源）
- 最近运行日志（状态检查/签到/抽奖）
- 功能开关状态

### 方式2：夸克内入口查看

进入夸克“设置/关于/我的”等页面，若注入成功会出现：

- `✅ 夸克自动签到模块（已注入）`

点击后可查看模块弹窗状态与日志。

### 方式3：ADB 查看

```bash
adb logcat -s QuarkAutoSign:D *:S
```

## 功能开关

在模块主页可直接切换：

- `自动签到`
- `每日自动抽奖（3次）`
- `Toast 提示`

说明：

- 关闭 `自动签到` 后只保留日志与注入提示
- 关闭 `自动抽奖` 后不再尝试抽奖
- 关闭 `Toast` 后仅写日志，不弹窗

## 工作流程

```text
夸克启动
  -> Hook Application.onCreate 成功
  -> 记录状态日志 + Toast 提示
  -> 根据开关执行自动签到/自动抽奖
  -> 写入历史记录与运行日志
```

## 抽奖机制说明

- 每天最多执行一轮（3 次）
- 采用“候选方法反射触发”策略，适配混淆版本
- 若当天接口签名变化，日志会显示“未找到可用方法/触发失败”

> 如果抽奖连续失败，把模块日志贴出来，可继续定向适配你当前夸克版本。

## 常见问题

### 1) 看不到 Hook 成功 Toast

- 检查模块是否在 NPatch/FPA 启用
- 检查是否重新注入了夸克
- 检查模块内 `Toast 提示` 开关是否关闭

### 2) 夸克里没有“已注入”入口

- 先进入夸克设置/个人页面再观察
- 有些页面布局动态加载，返回后再次进入
- 若仍无入口，先看模块首页日志是否有 Hook 记录

### 3) 签到没触发

- 看日志是否显示“今日已签到，跳过签到”
- 确认账号已登录、网络可用
- 若日志出现 `CameraCheckInManager 为 null`，可能需要适配新版本

### 4) 抽奖没有执行成功

- 看日志里每次抽奖的触发结果
- 若一直“未找到可用方法”，需要按当前版本调整方法名
