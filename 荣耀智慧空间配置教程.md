# 夸克扫描王自动签到 - 荣耀智慧空间配置教程

## 方案概述

**荣耀智慧空间（YOYO建议/场景联动）+ Xposed模块** 实现全自动签到：

```
每天 8:00（息屏状态）
    ↓
荣耀智慧空间触发 → 打开「夸克扫描王」APP
    ↓
Xposed模块检测APP启动 → 自动执行签到（3秒延迟）
    ↓
智慧空间等待15秒 → 自动返回桌面
    ↓
夸克APP进入后台 → MagicOS一段时间后清理
    ↓
第二天 8:00 再次触发...
```

**全程无需人工操作，息屏状态下完成**

> **设计说明**：Xposed模块只负责「签到」逻辑，不负责「返回桌面」。
> 这样可以避免影响你**正常使用夸克扫描王**——手动打开APP时，只会静默完成签到，不会强制跳转回桌面。

---

## 前置条件

1. **荣耀手机**（MagicOS 7.0/8.0/9.0）
2. **NPatch 或 FPA** 已安装并激活
3. **Xposed模块**（QuarkAutoSign）已启用
4. **夸克扫描王APP** 已安装（原版，NPatch修补版）
5. **已登录**夸克扫描王账号

## 第一部分：配置Xposed模块

### 步骤1：在NPatch/FPA中激活模块

**NPatch操作**：
```
1. 打开 NPatch 管理器
2. 点击底部「模块」标签
3. 找到「夸克扫描王自动签到」模块
4. 勾选启用开关
5. 点击「安装/更新」注入框架
6. 重启手机
```

**FPA操作**：
```
1. 打开 FPA 管理器
2. 进入「模块管理」
3. 找到「夸克扫描王自动签到」模块
4. 点击「启用」
5. 重新注入目标APP
```

### 步骤2：对夸克扫描王进行注入

```
1. 在 NPatch/FPA 首页
2. 点击「添加应用」或「选择应用」
3. 选择「夸克扫描王」
4. 点击「注入」或「修补」
5. 等待处理完成
6. 卸载原版夸克扫描王
7. 安装NPatch生成的修补版APK
```

### 步骤3：测试Xposed模块

```bash
# 连接电脑，执行：
adb logcat -s QuarkAutoSign:D *:S
```

**手动打开夸克扫描王APP**，预期输出：
```
QuarkAutoSign: QuarkAutoSign module loaded for: com.quark.scank
QuarkAutoSign: App started, check sign-in
QuarkAutoSign: Triggering sign-in...
QuarkAutoSign: Sign-in triggered OK
QuarkAutoSign: Auto-return to home screen
```

## 第二部分：配置荣耀智慧空间

### 入口位置

根据不同MagicOS版本，入口可能略有不同：

**MagicOS 7.x/8.x**：
```
设置 → 智慧助手 → 智慧空间 → 我的场景 → 新建场景
```

**MagicOS 9.x**：
```
设置 → 智慧助手 → YOYO建议 → 场景联动 → 新建场景
```

**桌面快捷入口**（部分机型）：
```
桌面 → 双指捏合 → 卡片 → 搜索「智慧空间」→ 添加到桌面
```

### 创建自动签到场景

#### 步骤1：新建场景

```
1. 打开「智慧空间」/「YOYO建议」
2. 点击「+」或「新建场景」
3. 场景名称输入：「夸克自动签到」
4. 点击「添加触发条件」
```

#### 步骤2：设置触发条件

```
触发条件类型：「定时」
    ↓
时间设置：每天 08:00（建议避开高峰期，如7:30或8:30）
    ↓
重复：每天
    ↓
开启「息屏执行」（关键！确保息屏状态下也能触发）
    ↓
点击「完成」
```

> **重要**：必须勾选「息屏执行」或「允许锁屏时执行」，否则手机息屏时不会触发。

#### 步骤3：设置执行任务（关键！）

必须添加两个任务：

```
执行任务1：「打开应用」
    ↓
选择应用：「夸克扫描王」
    ↓
点击「完成」

执行任务2：「等待」（必须在打开应用之后！）
    ↓
等待时长：15 秒（给签到和网络请求足够时间）
    ↓
点击「完成」

执行任务3：「返回桌面」或「关闭应用」（可选但建议）
    ↓
选择「返回桌面」或「关闭应用」
    ↓
点击「完成」
```

> **重要说明**：Xposed模块只负责在APP启动后自动执行签到，不会自动返回桌面。
> 如果你希望签完后自动返回桌面，必须在智慧空间场景中添加「等待15秒 → 返回桌面」任务。
> 如果不添加返回桌面任务，夸克APP会停留在前台，需要手动关闭。

#### 步骤4：保存场景

```
1. 点击右上角「保存」
2. 确认场景状态为「已启用」
3. 建议点击「测试」手动触发一次，验证流程
```

## 第三部分：MagicOS保活设置（重要！）

智慧空间的定时任务也可能被系统优化杀掉，需要以下设置：

### 1. 电池优化白名单

```
设置 → 电池 → 更多电池设置 → 电池优化
    ↓
找到「智慧空间」→ 设为「不允许」
找到「YOYO建议」→ 设为「不允许」（如果存在）
找到「夸克扫描王」→ 设为「不允许」
```

### 2. 后台应用管理

```
设置 → 应用 → 应用启动管理
    ↓
找到「智慧空间」→ 设为「手动管理」→ 允许自启动 + 允许后台活动
找到「夸克扫描王」→ 设为「手动管理」→ 允许自启动 + 允许后台活动
```

### 3. 休眠时保持网络连接

```
设置 → 电池 → 更多电池设置
    ↓
开启「休眠时保持网络连接」
（确保息屏时网络可用，签到需要联网）
```

### 4. 锁屏清理保护

```
最近任务 → 找到「智慧空间」（如果可见）→ 下拉锁定
```

## 常见问题

### Q1: 智慧空间定时任务没有触发？

**检查项**：
- [ ] 场景是否已启用？
- [ ] 是否勾选了「息屏执行」？
- [ ] 电池优化是否关闭了？
- [ ] 是否允许了自启动？

**MagicOS限制**：部分机型限制每天定时任务数量，检查是否有其他场景冲突。

### Q2: APP打开了但没有签到？

**检查项**：
- [ ] Xposed模块是否已启用？
- [ ] 夸克扫描王是否是NPatch修补版？
- [ ] 是否已登录夸克账号？
- [ ] 查看日志：`adb logcat -s QuarkAutoSign:D`

### Q3: 签到成功了但没有自动返回桌面？

**正常现象**：
- autoFinish() 使用 Handler.postDelayed，部分MagicOS版本可能限制后台Activity跳转
- 夸克APP留在前台也不影响，MagicOS会稍后清理
- 如果介意，可以在智慧空间场景中添加「等待15秒 → 返回桌面」任务

### Q4: 连续几天签到失败？

**可能原因**：
1. **Token过期**：Xposed模块调用原生签到接口，依赖APP本地登录状态
   - 解决：定期手动打开一次夸克APP保持登录
2. **网络问题**：息屏时WiFi断开
   - 解决：设置 → WLAN → 高级设置 → 休眠时保持WLAN连接
3. **夸克服务器风控**：连续自动签到被检测
   - 解决：将定时时间改为不固定（如周一8:00、周二8:15等）

### Q5: 智慧空间和Xposed模块的时间不一致？

**建议**：
- 智慧空间触发时间：每天固定时间（如8:00）
- Xposed模块：每次APP启动时检查当天是否已签到，防止重复
- 两者配合，不冲突

## 高级设置

### 设置多个备份触发时间

为防止某个时间点系统未触发，可以创建多个场景：

| 场景名称 | 触发时间 | 作用 |
|----------|----------|------|
| 夸克签到-主 | 08:00 | 主要签到 |
| 夸克签到-备1 | 12:00 | 午休补签 |
| 夸克签到-备2 | 20:00 | 晚间补签 |

Xposed模块会检查当天是否已签到，不会重复签到。

### 只在工作日签到

```
智慧空间场景 → 编辑触发条件
    ↓
定时：周一到周五 08:00
    ↓
保存
```

### 签到后发送通知

```
智慧空间场景 → 添加执行任务
    ↓
执行任务：「发送通知」
    ↓
标题：「夸克签到完成」
内容：「今日签到已自动完成」
```

## 方案优势总结

| 优势 | 说明 |
|------|------|
| 无需改APK | 保留原版签名，随时更新APP |
| 免Root | NPatch/FPA免Root方案 |
| 系统级触发 | 荣耀智慧空间是系统组件，保活优先级高 |
| 息屏执行 | 支持锁屏状态下触发 |
| 防重复 | Xposed模块本地记录，一天只签一次 |
| 自动清理 | 签到后返回桌面，不占用前台 |
