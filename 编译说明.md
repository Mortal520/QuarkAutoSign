## 方案一：使用 Android Studio（推荐）

### 步骤1：准备环境

1. 下载并安装 [Android Studio](https://developer.android.com/studio)
2. 安装 SDK Platform 34（或最新版）

### 步骤2：导入项目

```
1. 打开 Android Studio
2. 选择 "Open an Existing Project"
3. 选择本文件夹（Xposed_自动签到模块）
4. 等待 Gradle Sync 完成
```

### 步骤3：编译APK

```
1. 点击菜单栏 Build → Build Bundle(s) / APK(s) → Build APK(s)
2. 等待编译完成
3. 输出路径：app/build/outputs/apk/debug/app-debug.apk
```

### 步骤4：重命名（可选）

```
将 app-debug.apk 重命名为 QuarkAutoSign-v1.0.apk
```

## 方案二：使用命令行 Gradle

### 前置条件

- 安装 Java JDK 11+（设置 JAVA_HOME 环境变量）
- 下载 Android SDK Command Line Tools

### 步骤1：设置环境变量

```bash
# Windows PowerShell
$env:ANDROID_HOME = "C:\Users\你的用户名\AppData\Local\Android\Sdk"
$env:PATH += ";$env:ANDROID_HOME\cmdline-tools\latest\bin"
```

### 步骤2：编译

```bash
# 进入项目目录
cd Xposed_自动签到模块

# 使用 Gradle Wrapper 编译（如果有）
gradlew assembleDebug

# 或直接使用系统 Gradle
gradle assembleDebug
```

### 步骤3：输出

```
输出路径：build/outputs/apk/debug/Xposed_自动签到模块-debug.apk
```

## 方案三：手动编译（最小依赖）

如果无法安装 Android Studio，可以手动编译：

### 需要下载的工具

1. **Android SDK Command Line Tools**
   - 下载：https://developer.android.com/studio#command-tools
   
2. **Xposed API JAR**
   - 下载：https://repo1.maven.org/maven2/de/robv/android/xposed/api/82/api-82.jar
   
3. **Android Platform 34 JAR**
   - 路径：`$ANDROID_HOME/platforms/android-34/android.jar`

### 编译脚本（Windows）

创建 `build.bat`：

```batch
@echo off
setlocal

:: 配置路径（根据实际路径修改）
set ANDROID_HOME=C:\Users\%USERNAME%\AppData\Local\Android\Sdk
set ANDROID_JAR=%ANDROID_HOME%\platforms\android-34\android.jar
set XPOSED_JAR=api-82.jar
set BUILD_TOOLS=%ANDROID_HOME%\build-tools\34.0.0

:: 创建输出目录
mkdir build\intermediates\classes 2>nul
mkdir build\outputs 2>nul

:: 编译 Java 文件
javac -source 1.8 -target 1.8 -cp "%ANDROID_JAR%;%XPOSED_JAR%" -d build\intermediates\classes src\main\java\com\qurk\autosign\QuarkAutoSign.java

:: 打包成 DEX
%BUILD_TOOLS%\d8 --release --output build\intermediates\dex build\intermediates\classes\com\qurk\autosign\*.class

:: 打包成 APK（需要 apksigner 或 jarsigner）
:: 这里需要额外步骤...

echo Build complete!
pause
```

**注意**：手动编译较复杂，推荐使用 Android Studio。

## 编译常见问题

### Q1: Gradle Sync 失败，提示找不到 Xposed API？

**解决**：在 `build.gradle` 中确认依赖配置：
```gradle
dependencies {
    compileOnly 'de.robv.android.xposed:api:82'
}
```

如果网络问题导致无法下载，可以手动下载 `api-82.jar` 放入 `libs` 目录：
```gradle
dependencies {
    compileOnly files('libs/api-82.jar')
}
```

### Q2: 编译报错 "找不到符号 XposedHelpers"？

**解决**：确保 `api-82.jar` 正确添加到依赖中。Xposed API 包含 `de.robv.android.xposed` 包。

### Q3: 编译通过但模块无法生效？

**检查项**：
- [ ] `xposed_init` 文件是否在 `assets` 目录？
- [ ] `xposed_init` 内容是否为 `com.qurk.autosign.QuarkAutoSign`？
- [ ] `AndroidManifest.xml` 中是否包含 Xposed meta-data？
- [ ] NPatch/FPA 中是否正确注入到夸克扫描王？

### Q4: 如何生成 Release 版本？

```
Android Studio: Build → Generate Signed Bundle / APK → APK
选择或创建签名密钥
选择 release
生成 QuarkAutoSign-release.apk
```

## 编译输出文件

| 文件 | 大小 | 用途 |
|------|------|------|
| `app-debug.apk` 或 `Xposed_自动签到模块-debug.apk` | 约 50-100 KB | 调试版本，可直接安装测试 |
| `app-release.apk` | 约 30-80 KB | 发布版本，需签名 |

**模块APK非常小**，因为只包含 Xposed Hook 代码，不依赖APP本身。
