# 固定签名配置说明

## 已生成签名文件

- **文件名**: `release.keystore`
- **密钥别名**: `quarkautosign`
- **密钥密码**: `QuarkAutoSign2024`
- **有效期**: 10000 天（约 27 年）

## GitHub Actions 自动构建配置

### 步骤 1: 配置 Secrets

在 GitHub 仓库页面：
`Settings → Secrets and variables → Actions → New repository secret`

添加以下 4 个 Secrets：

| Secret 名称 | 值 | 说明 |
|------------|-----|------|
| `KEYSTORE_BASE64` | (见下方) | 签名文件的 base64 编码 |
| `KEYSTORE_PASSWORD` | `QuarkAutoSign2024` | Keystore 密码 |
| `KEY_ALIAS` | `quarkautosign` | 密钥别名 |
| `KEY_PASSWORD` | `QuarkAutoSign2024` | 密钥密码 |

### KEYSTORE_BASE64 获取方式

在本地项目目录执行：
```powershell
[Convert]::ToBase64String([System.IO.File]::ReadAllBytes("release.keystore")) | Set-Clipboard
```

然后粘贴到 GitHub Secrets 的 `KEYSTORE_BASE64` 中。

### 步骤 2: 触发构建

配置完成后，推送代码或手动触发 workflow：
```
Actions → Build Xposed Module → Run workflow
```

构建产物将使用固定签名，可覆盖安装。

## 本地构建配置

如需本地构建使用固定签名，创建 `keystore.properties` 文件：

```properties
storeFile=release.keystore
storePassword=QuarkAutoSign2024
keyAlias=quarkautosign
keyPassword=QuarkAutoSign2024
```

然后执行：
```bash
./gradlew assembleRelease
```

## 注意事项

1. **不要提交 `release.keystore` 到 GitHub**（已加入 .gitignore）
2. **妥善保管签名密码**，丢失后无法找回
3. 同一签名构建的 APK 可直接覆盖安装，无需卸载
4. 如需更换签名，必须先卸载旧版本再安装新版本

## 版本号递增

每次发布前建议递增 `build.gradle` 中的 `versionCode`：

```gradle
defaultConfig {
    versionCode 2  // 每次发布 +1
    versionName "1.1"
}
```
